# ARCHITECTURE.md

# System Architecture

## Overview

The Agentic Multi-Modal Processing System follows a **microservices-inspired architecture** with clear separation of concerns. The system is designed to be scalable, maintainable, and production-ready.

## Components

### 1. Frontend Layer (React)
**Responsibility**: User interface and interaction

**Key Components**:
- Chat interface for conversation-style interaction
- File upload with drag-and-drop
- Real-time message streaming
- Error handling and user feedback

**Technology Stack**:
- React 18.2
- Axios for HTTP requests
- Lucide React for icons
- CSS for styling

### 2. API Layer (FastAPI)
**Responsibility**: Request routing and orchestration

**Endpoints**:
- `POST /api/process` - Main processing endpoint
- `GET /api/health` - Health check
- `POST /api/estimate-cost` - Cost estimation (bonus)

**Features**:
- CORS handling
- File validation
- Error handling
- Logging

### 3. Intent Detection Service
**Responsibility**: Analyze input and determine task type

**Algorithm**:
1. Keyword matching for explicit intents
2. Pattern matching for code detection
3. URL detection for YouTube
4. Fallback to clarification when ambiguous

**Intents Supported**:
- Summarization
- Sentiment Analysis
- Code Explanation
- Action Items
- YouTube Transcript
- Conversational
- Needs Clarification

### 4. File Processing Service
**Responsibility**: Extract content from various file types

**Processors**:
- **Image Processor**: Tesseract OCR
- **PDF Processor**: PyPDF2 + OCR fallback
- **Audio Processor**: OpenAI Whisper

**Features**:
- Async processing
- Error handling
- Confidence scoring
- Format detection

### 5. Task Execution Service
**Responsibility**: Execute tasks based on detected intent

**Handlers**:
- Summary Handler: 3-format output
- Sentiment Handler: Label + confidence + justification
- Code Handler: Purpose, logic, bugs, complexity
- Action Items Handler: Structured task list
- Clarification Handler: Ask questions
- Conversational Handler: General Q&A

**Integration**: Claude Sonnet 4 via Anthropic API

## Data Flow

```
User Input → Frontend → API Gateway → Intent Detector
                                            ↓
                                    File Processor (if file)
                                            ↓
                                    Task Executor → Claude AI
                                            ↓
                                    Response Formatter
                                            ↓
                                    Frontend Display
```

## Scalability Considerations

### Horizontal Scaling
- Stateless API design
- Load balancer ready
- Database for session management (future)

### Vertical Scaling
- Async file processing
- Task queuing system (future)
- Caching layer (future)

### Performance Optimization
- Lazy loading of ML models
- File size limits
- Request rate limiting
- Response streaming (future)

## Security

### Input Validation
- File type checking
- File size limits (10MB)
- Content sanitization
- SQL injection prevention (N/A - no DB)

### API Security
- API key management
- CORS configuration
- Rate limiting (future)
- HTTPS in production

### Data Privacy
- No persistent storage of user data
- Temporary file cleanup
- No logging of sensitive content

## Monitoring & Logging

### Logging Levels
- INFO: Normal operations
- WARNING: Recoverable errors
- ERROR: Failures requiring attention
- DEBUG: Development info

### Metrics (Future)
- Request latency
- Success/failure rates
- API costs
- File processing times

## Future Enhancements

### Multi-Agent Orchestration (Bonus)
```
Planner Agent → Task Queue → Executor Agent(s) → Aggregator
```

### Cost Optimization
- Request caching
- Response streaming
- Model selection based on complexity

### Advanced Features
- Multi-language support
- Custom model fine-tuning
- Batch processing
- WebSocket support for real-time updates
